🏗️ 4. System Analysis & Design

4.1 Problem Statement & Objectives

Problem Statement

Heart disease remains the leading cause of death worldwide. Early risk detection using routine health data can save lives, but traditional screening methods are costly, time-consuming, and require specialized clinical expertise. There is a need for an automated, explainable, and accessible system that can predict heart disease risk from commonly available patient data while providing transparent reasoning that clinicians can trust.

Project Objectives

  1. Develop a high-accuracy binary classification model (target: ROC-AUC ≥ 0.82) using the CDC BRFSS dataset (319,795 patients).
  2. Engineer 25+ medically meaningful features from 18 raw columns to enhance model performance.
  3. Provide Explainable AI (SHAP) so doctors understand why a prediction was made, not just what it is.
  4. Build an interactive web application (Streamlit) for real-time patient risk assessment.
  5. Design a Star Schema data warehouse for analytical reporting via Power BI.
  6. Segment patients into risk clusters (K-Means) for population health management.

Level 1 — Detailed Data Flow

1.0 Data Ingestion — Raw CSV (319K × 18) → Python pandas DataFrame

2.0 Feature Engineering — Raw 18 cols → 43+ engineered features (Age_Numeric, BMI_Category, Risk_Score, Lifestyle_Score, Geographic Enrichment, etc.)

3.0 Star Schema Builder — Enriched data → Split into 6 dimension tables + 1 fact table → Export to Excel/SQL

4.0 ML Training Pipeline — Encoded features → SMOTE (train only) → Train 6 models → Optuna tuning → Select best (LightGBM V2)

5.0 Prediction & Clustering — Best model → Predict 319K patients → K-Means clustering → Inject ML_Probability, ML_Prediction, Patient_Segment into fact table

6.0 Presentation — Streamlit app (real-time prediction + SHAP) + Power BI dashboards (KPIs, maps, segments)

4.6 Activity Diagram — Patient Risk Assessment Flow

● [Start]

→ Doctor opens Streamlit app

→ Doctor selects Model Version (V1 or V2)

→ Doctor enters patient vitals (Age, BMI, Smoking, Diabetes, Stroke, etc.)

→ Doctor clicks "Calculate Risk"

→ System loads selected model (.pkl) and scaler

→ System applies feature encoding & scaling

→ System computes ML probability

→ [Decision] Probability ≥ Youden threshold (49.22%)?

   ├── YES → Display 🔴 POSITIVE card + risk percentage

   └── NO → Display 🟢 NEGATIVE card + risk percentage

→ System computes SHAP values for this patient

→ System renders SHAP waterfall plot

→ Doctor reviews explanation and makes clinical decision

● [End]
