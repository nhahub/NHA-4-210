7.1 User Manual (Streamlit Web App)

The Streamlit web application is designed to be highly user-friendly for medical professionals. Here is a step-by-step guide to using the predictor:

  1. Access the Application: Launch the application locally using streamlit run "Machine Learning/app.py" and navigate to http://localhost:8501.
  2. Select Model Version: In the left sidebar, choose the model version:
    • V1 (Baseline): The initial model version.
    • V2 (Optimized) [Recommended]: The Optuna-tuned LightGBM model which offers higher recall and overall accuracy.
  3. Enter Patient Metrics: Use the interactive form inputs in the sidebar:
    • Select demographic parameters (Age range, Sex, Race).
    • Adjust body parameters (BMI slider, General Health rating).
    • Select habits (Smoking, Alcohol, Physical Activity).
    • Check chronic medical conditions (Stroke, Diabetes, Kidney Disease, Asthma, Skin Cancer, Difficulty Walking).
  4. Calculate Risk: Click the "Calculate Risk" button.
  5. Interpret Results:
    • Risk Prediction Card: Renders a colored indicator. 🟢 NEGATIVE indicates a low risk, while 🔴 POSITIVE warns of heart disease risk (based on the optimal Youden threshold).
    • SHAP Waterfall Plot: Visualizes factors contributing to this patient's prediction. Red bars increase risk, while blue bars decrease it.
    • Clinical Advice: Review the automatically generated lifestyle recommendations tailored to the patient's habits.

7.2 Technical Documentation Summary

System Architecture Summary

  • Data Prep: 25+ features engineered from 18 raw columns, followed by standard scaling and one-hot encoding.
  • Model: Tuned LightGBM classifier optimized on Stratified K-Folds, achieving 0.8404 ROC-AUC and 80.57% Recall.
  • Explainable AI: Integrates SHAP for local and global model explanations.
  • Data Warehouse: Star Schema consisting of 1 central fact table (fact_Diagnosis) and 6 dimensions (dim_Patients, dim_HealthStatus, dim_Conditions, dim_Lifestyle, dim_Geography, dim_Date).
  • BI Tool: Power BI loaded with the Star Schema, displaying KPIs, geographic maps, and patient segments.

Data Preprocessing Details

  • Missing Values: Filled 192,255 missing values in Comorbidity_Level with 'None' where Comorbidity_Count = 0.
  • Class Imbalance: Treated using SMOTE on training folds combined with algorithm-specific weights (e.g. scale_pos_weight=10.68).
  • Encoding: Binary variables mapped to 0/1, ordinal scales normalized, and categorical categories one-hot encoded.

7.3 Project Presentation Summary

Challenges faced and resolved

  1. Severe Class Imbalance: An imbalanced 1:10.7 positive-to-negative ratio was solved by using Stratified K-Fold splits, SMOTE, and tuning the decision threshold via Youden's J statistic to ensure high recall for screening.
  2. Overfitting in Deep Learning: Our Multilayer Perceptron (MLP) overfit on SMOTE patterns (0.94 train vs 0.76 test AUC). This was addressed by selecting LightGBM V2 with strict regularization.
  3. Large Datasets: Processing 319,795 rows in Power BI and SQL Server. Solved by designing an efficient Star Schema database in SQL Server to optimize query execution and report rendering.

Key Project Outcomes

  • Achieved 80.57% Recall in predicting heart disease risk, establishing clinical reliability.
  • Integrated Explainable AI (SHAP) to bridge the gap between machine learning and medical trust.
  • Connected AI predictions directly into a Data Warehouse, allowing Power BI dashboards to segment patients based on live machine learning classifications.
