3.2 User Stories & Use Cases

UC-01: Doctor Performs Real-Time Patient Risk Assessment

Actor: Clinician / Doctor

Precondition: Streamlit app is running; patient vital signs are available

Main Flow:

  1. Doctor opens the Streamlit web application
  2. Doctor selects model version (V1 Baseline or V2 Optimized)
  3. Doctor enters patient data (Age, BMI, Smoking status, Diabetes, Stroke history, etc.)
  4. Doctor clicks "Calculate Risk"
  5. System computes ML probability using the selected model
  6. System applies Youden's optimal threshold (49.22% for V2)
  7. System displays: Risk probability, POSITIVE/NEGATIVE diagnosis card, SHAP waterfall plot

Postcondition: Doctor has a clear diagnosis with explainable AI reasoning for clinical decision

UC-02: Hospital Admin Analyzes Patient Population via Power BI

Actor: Hospital Manager / Data Analyst

Precondition: Power BI desktop installed; .pbix file loaded with Star Schema data

Main Flow:

  1. Admin opens the Power BI dashboard
  2. Admin uses slicers to filter by State, Age Group, Risk Tier, or Patient Segment
  3. Dashboard updates KPIs: total patients, heart disease rate, average risk score
  4. Admin drills down into geographic distribution maps
  5. Admin reviews K-Means patient segmentation clusters

Postcondition: Admin has actionable insights for resource allocation and preventive programs

UC-03: Data Analyst Queries Star Schema via SQL

Actor: Data Analyst / Data Engineer

Main Flow:

  1. Analyst connects to SQL Server database
  2. Analyst runs queries joining fact_Diagnosis with dimension tables
  3. Analyst extracts demographic insights (e.g., heart disease rate by state, age, BMI category)
  4. Analyst exports results for reporting

UC-04: Data Analyst Runs ML Pipeline for Retraining

Actor: ML Engineer

Main Flow:

  1. Engineer updates the dataset CSV in the data/ folder
  2. Engineer runs python pipelines/ml_pipeline_v2.py
  3. Pipeline trains 6 models with Optuna optimization
  4. Pipeline saves best model, metrics, and visualization outputs
  5. Engineer updates the Streamlit app with new model artifacts
